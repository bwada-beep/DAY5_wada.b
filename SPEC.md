# **SPEC.md**

## **1\. 概要**

本アプリケーションは、音声ファイルを入力として講義内容の文字起こし・要約・要点抽出を行うフロントエンドアプリである。  
ユーザーはAPIキー（Gemini API）を入力し、音声ファイルをアップロードすることで処理を実行できる。  
本アプリは **HTML / CSS / JavaScript のみで構成し、GitHub Pages 上で公開する前提** とする。

---

## **2\. 画面の流れ（ユーザー導線）**

1. APIキーを入力・保存（初回のみ必須）  
   * localStorageに保存  
2. 講義名を入力する（任意・推奨）  
3. 音声ファイルを選択してアップロード  
   * 将来的に録音機能の追加を想定  
4. 「実行」ボタンを押す  
5. 以下の結果が表示される  
   * 文字起こし  
   * 概要（3行）  
   * 要点（3〜5個の箇条書き）  
6. 各出力結果はコピー可能

---

## **3\. 入力仕様**

### **必須入力**

* APIキー（Gemini API）  
* 音声ファイル

### **任意入力**

* 講義名

### **音声ファイル仕様**

* 最大サイズ：10MB  
* 対応形式：mp3 / wav / m4a  
* 検証用条件：**短い音声（30秒〜2分）を前提に利用すること**

---

## **4\. 出力仕様**

### **① 文字起こし**

* 音声内容をテキスト化

### **② 概要**

* **必ず3行で要約する**

### **③ 要点**

* **3〜5個の箇条書きで出力する**

### **UI要件**

* 各出力にコピーボタンを設置  
* ワンクリックでコピー可能

---

## **5\. エラー処理**

### **APIキー未設定**

* 「APIキーを入力してください」

### **音声ファイル未選択**

* 「音声ファイルを選択してください」

### **API通信失敗**

* 「処理中にエラーが発生しました。再度お試しください」

### **ファイルサイズ超過**

* 「ファイルサイズは10MB以下にしてください」

---

## **6\. 技術仕様**

### **使用API**

* Gemini API（Google）  
  * 音声 → テキスト  
  * テキスト → 要約・要点抽出

### **アーキテクチャ**

* フロントエンドのみ（HTML / CSS / JavaScript）  
* バックエンドは使用しない  
* GitHub Pagesで公開

### **APIキー管理**

* ユーザーが画面上で入力  
* localStorageに保存  
* ソースコードへの直書きは禁止  
* 公開リポジトリへの保存は禁止

---

## **8\. 重要ポイント**

* APIキーはユーザー入力＋ローカル保存で管理  
* フロントエンドのみで完結する設計  
* ファイルサイズ制限（10MB）を事前チェック  
* 短時間音声での検証を前提とする  
* 非同期処理（ローディング表示）を実装  
* ユーザーに分かりやすいエラーメッセージを表示  
* コピーしやすいUI設計
